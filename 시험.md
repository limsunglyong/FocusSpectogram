# 지침

1. 모든 시험 내용은 반드시 앱 버전을 기록한다.
2. 모든 시험 완료된 내용은 시험-아카이브.md에 추가하고 시험.md에서는 삭제처리 한다.
3. "2. 시험 항목"은 자동 시험, 수동 시험 으로 구분하여 표시한다.
   - 자동 시험 : 스스로 할 수 있는 시험 항목을 나열하고 시험 계획을 간략히 기록한 다음, 시험 결과를 사용자에서 보여준다. 이후 다음 작업시 자동 시험 내용은 리셋하고 새로운 내용으로 변경한다.
   - 수동 시험 : 사용자가 해야하는 시험 항목을 리스트로 표시한다. 섹션 "2. 시험 항목" 에서는 간략한 시험 항목 번호와 시험 내용, 판정 기준을 간략하게 표로 보여준다. 구체적인 시험 내용에 대해서는 섹션 "3. 시험 방법 및 절차"에 정리하되, 어떤 것을 시험하는 것이며 어떤 순서로 시험하는 것인지 step-by-step으로 기록한다.
4. 앱 실행과 관련된 일반적인 정보들은 섹션 "1. 시험 개요" 부분에 추가하여 작성한다.

## 3. 시험 방법 및 절차

### M8.1-1 : Hop Size 드롭다운
1. `npm.cmd run dev -- --host 127.0.0.1` 로 앱을 실행한다.
2. 오디오 파일을 로드해 스펙트로그램 분석이 완료될 때까지 기다린다.
3. FFT Properties 패널에서 Hop Size를 512에서 256 또는 1024로 변경한다.
4. 상태 배지가 ANALYZING으로 바뀐 뒤 STABLE/DONE 상태로 돌아오는지 확인한다.
5. 하단 요약의 Δt(ms)가 Hop Size에 맞게 바뀌는지 확인한다.

### M8.1-2 ~ M8.1-4 : LUFS 기준 시각화
1. LUFS Level을 -23 LUFS로 둔 상태에서 LUFS Plane 체크박스를 켠다.
2. 3D 씬에 반투명 적색 수평 평면이 나타나는지 확인한다.
3. 세로 축 제목이 LEVEL (LUFS)로 보이고, 선택한 LUFS 라벨이 적색으로 표시되는지 확인한다.
4. LUFS Level을 -14, -30 등으로 바꿔 평면 높이와 라벨이 함께 이동하는지 확인한다.
5. LUFS Plane이 on일 때 기준을 넘는 표면 영역이 적색으로 강조되는지 확인한다.
6. LUFS Plane 체크를 해제하면 평면과 적색 초과 강조가 사라지는지 확인한다.

### M8.1-5 : 회귀 확인
1. 재생/일시정지/seek를 수행해 플레이헤드가 정상 이동하는지 확인한다.
2. Viewport Settings의 Played Region SHOW/FADE/HIDE가 기존처럼 동작하는지 확인한다.
3. Perspective ISO/ORTHO/3D 전환 및 Reset View가 기존처럼 동작하는지 확인한다.

## 4. 주의 사항

- 현재 LUFS Plane은 STFT dB 레벨을 LUFS 기준선에 대응시킨 시각 비교 기능이다.
- 방송 표준의 정확한 integrated/short-term/momentary LUFS 계산은 별도 loudness DSP 기능으로 추가해야 한다.
