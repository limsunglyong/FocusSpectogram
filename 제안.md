# 제안: Reference Tone Match / Auto EQ 보조 기능

## 배경

- 사용자가 현재 곡에서 EQ를 조정했을 때 매우 마음에 드는 톤을 얻었다.
- 하지만 동일한 EQ 설정을 다른 곡에 그대로 적용하면 곡마다 원본 스펙트럼, 악기 구성, 믹스 상태가 달라 같은 느낌을 보장하기 어렵다.
- 따라서 단순 EQ preset 복사보다, 현재 곡에서 얻은 “톤 변화의 성격”을 저장하고 다른 곡에 맞게 자동 조정하는 기능이 필요하다.

## 핵심 개념

현재 곡 A에서:

1. 곡 A 원본 평균 스펙트럼을 계산한다.
2. 사용자가 조정한 EQ 응답 곡선을 계산한다.
3. 곡 A에 적용된 톤 변화 프로파일을 저장한다.

다른 곡 B에서:

1. 곡 B의 평균 스펙트럼을 계산한다.
2. 저장된 Reference Tone 변화 프로파일과 비교한다.
3. 곡 B가 비슷한 톤 인상을 갖도록 5밴드 EQ 값을 자동 추정한다.
4. 사용자는 결과를 듣고 3D EQ 핸들/우측 패널로 미세 조정한다.

## 왜 단순 preset 복사로는 부족한가

- 곡 A에서 고역 +3dB가 좋았더라도, 곡 B는 이미 고역이 강할 수 있다.
- 곡 A에서 저역을 올린 이유가 원본 저역 부족 때문이었다면, 저역이 이미 많은 곡 B에는 같은 boost가 과할 수 있다.
- 따라서 “같은 EQ 값”보다 “비슷한 최종 톤 인상”을 목표로 해야 한다.

## 추천 구현 단계

### 1차: EQ Preset 저장/불러오기

- 현재 `eqBands`를 이름과 함께 저장한다.
- 다른 곡에서 그대로 불러올 수 있게 한다.
- 장점: 구현이 쉽고, 기준점으로 유용하다.
- 한계: 곡별 차이를 보정하지 못한다.

### 2차: Reference Tone 저장

- 현재 곡의 평균 스펙트럼과 현재 EQ 응답을 기반으로 “Tone Delta”를 저장한다.
- Tone Delta는 “이 곡에서 사용자가 좋아한 주파수별 변화”를 의미한다.
- 저장 데이터 예:
  - reference name
  - source average spectrum
  - EQ response curve
  - tone delta curve
  - original eqBands

### 3차: Auto Match

- 다른 곡에서 `Match Current Track to Reference Tone` 버튼을 제공한다.
- 현재 곡 평균 스펙트럼을 계산한다.
- 저장된 Tone Delta를 참고해 5밴드 EQ gain을 자동 조정한다.
- 1차 Auto Match는 frequency/Q까지 건드리지 않고 gain만 조정하는 것을 추천한다.
- 이유:
  - 안정적이고 예측 가능하다.
  - 사용자가 3D EQ 핸들로 후속 조정하기 쉽다.
  - 과도한 자동 최적화로 인한 이상한 EQ 곡선을 피할 수 있다.

### 4차: 고급 매칭

- frequency/Q까지 자동 최적화한다.
- 곡 전체 평균이 아니라 선택 구간 기준으로 매칭한다.
- verse/chorus 등 섹션별 reference tone을 지원한다.
- LUFS, transient, stereo width, noise profile까지 함께 고려하는 마스터링 보조 기능으로 확장한다.

## UI 제안

### EQ 패널 또는 별도 Tone Match 패널

- `Save EQ Preset`
- `Load EQ Preset`
- `Save as Reference Tone`
- `Match to Reference`
- `Amount` 슬라이더
  - 0% = 적용 없음
  - 100% = 자동 추정 EQ 전체 적용
  - 50% = 절반만 적용

### 표시 방식

- 3D EQ 오버레이에 자동 매칭 목표 곡선을 얇은 보조선으로 표시할 수 있다.
- 현재 EQ 곡선과 Auto Match 목표 곡선을 서로 다른 색으로 비교하면 이해가 쉬울 수 있다.

## 알고리즘 초안

1. 현재 `Spectrogram`의 `data`에서 주파수 bin별 평균 dB를 계산한다.
2. 현재 EQ 밴드 응답을 `computeEqMagnitudeDb`로 계산한다.
3. Reference Tone 저장 시:
   - `referenceAvgDb[bin]`
   - `eqResponseDb[bin]`
   - `targetAvgDb[bin] = referenceAvgDb[bin] + eqResponseDb[bin]`
   - 또는 `toneDeltaDb[bin] = eqResponseDb[bin]`
4. 다른 곡에서 Auto Match 시:
   - `currentAvgDb[bin]` 계산
   - 목표 곡선과 현재 곡선의 차이를 계산
   - 5개 EQ 밴드 중심 주파수 근처의 평균 차이를 gain 후보로 변환
   - gain 값을 안전 범위로 clamp한다.
5. 적용 후 사용자가 수동 조정한다.

## 주의할 점

- “같은 느낌”은 스펙트럼만으로 100% 결정되지 않는다.
- 다이내믹, LUFS, 트랜지언트, 공간감, 왜곡, 리버브도 톤 인상에 영향을 준다.
- 따라서 이 기능은 완전 자동 마스터링이 아니라 “좋은 출발점 자동 제안”으로 설계하는 것이 좋다.
- 과도한 자동 보정 방지를 위해 gain 범위, smoothing, amount 조절이 필요하다.
